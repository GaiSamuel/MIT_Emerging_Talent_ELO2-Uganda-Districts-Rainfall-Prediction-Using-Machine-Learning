## 1. Level 0: Raw Source Data

*These files are the "Ground Truth" sources. They are large binary or spatial files.*

- **Climate Drivers (ERA5):**
  - **Format:** `.grib` (GRIB2)
  - **Source:** Copernicus Climate Change Service (C3S).
  - **Contents:** Monthly aggregates of Temperature, Pressure, Wind, etc.
- **Rainfall (CHIRPS):**
  - **Format:** `.tif` (GeoTIFF Raster)
  - **Source:** Climate Hazards Group (UCSB).
  - **Contents:** High-resolution satellite-station blended precipitation.
- **Geographic Boundaries:**
  - **Format:** `.shp` (Shapefile)
  - **Source:** Uganda Bureau of Statistics (UBOS) / UNOCHA.
  - **File:** `uga_admbnda_adm2_ubos_20200824.shp`
  - **Usage:** Used to perform Zonal Statistics (spatial aggregation) to convert
 gridded climate data into district averages.

## 3. Level 2: Clean Master Dataset (Training Data)

*This is the primary dataset used for Exploratory Data Analysis (EDA) and Model Training.*

- **File:** [`uganda_monthly_rainfall_and_climate_variables_1981_2025.csv`](https://github.com/GaiSamuel/MIT_Emerging_Talent_ELO2-Uganda-Districts-Rainfall-Prediction-Using-Machine-Learning/blob/main/1_datasets/clean_data/uganda_monthly_rainfall_and_climate_variables_1981_2025.csv)
- **Processing Steps:**
    1. Merged all Level 1 files on `District` and `Date`.
    2. Standardized units (Kelvin $\rightarrow$ Celsius).
    3. Imputed missing values (e.g., filled the temperature (1988-1999) and
    surface pressure (2024) gap using climatology).
    4. Removed "dead" districts with insufficient data.
